5/**
6 * Class for handling task completion triggers.
7 *
8 * Tasks that have smart or automatic completion are managed here.
9 * Each task type has:
10 * - Task path constants (TASK_PATHS) for single source of truth
11 * - Registration method that sets up both hooks and validators
12 * - Handler methods that respond to WordPress/plugin events
13 * - Validation methods that check existing site state on plan initialization
14 *
15 * Architecture:
16 * - Task paths are defined once in TASK_PATHS constant
17 * - Registration methods combine hook and validator setup
18 * - Handlers use mark_task_as_complete_by_path() with constants
19 * - Validators reuse the same logic as handlers for consistency
20 *
21 * File Organization:
22 * # Constructor & Setup
23 * # Product Tasks (WooCommerce)
24 * # Payment Tasks (WooCommerce)
25 * # Blog Tasks (Content Creation)
26 * # Jetpack Tasks (Performance & Stats)
